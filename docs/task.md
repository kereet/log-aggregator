# Техническое задание

Разработать микросервис на Go, который принимает файлы-логи, парсит их и агрегирует информацию, сохраняет полученные данные в PostgreSQL и предоставляет REST‑API для получения информации о сущностях.

## 1. Технологический минимум

- **Язык** — Go 1.22+ (можно использовать только стандартный пакет `net/http`).
- **База данных** — PostgreSQL (запуск через Docker Compose).

---

## 2. Функциональные требования

| №    | Функция                  | Что должно делаться                                                                                                                                                                                                                                                                                                                                                         |
|------|--------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| F-1  | Работа с файлом          | Доступ к файлу осуществляется через монтированную папку в самом проекте. Название папки должно быть `data/`.                                                                                                                                                                                                                                                               |
| F-2  | Парсинг                  | Разбить файлы на секции и собрать информацию по уникальным идентификаторам в объекты. Не все объекты могут быть полными. При любой ошибке отклонить файл и выкинуть ошибку.                                                                                                                                                                                                |
| F-3  | Агрегация топологии      | На основе `nodes` и `ports` построить граф:<br>• узлы (host / switch);<br>• порты.<br>Будет плюсом, если подумать, какие связи могли бы быть, и записать это в `README.md`.                                                                                                                                                                                                 |
| F-4  | Сохранение в БД          | Таблицы (минимум): `logs`, `nodes`, `ports`, `nodes_info`. Все связи реализованы через внешние ключи (FK). Также необходим скрипт создания базы данных или миграции. Они должны применяться автоматически при сборке.                                                                                                                                                       |
| F-5  | API-доступ к топологии   | 1. `POST /api/v1/parse/` – возвращает `log_id`. В теле передать путь до архива с логом.<br>2. `GET /api/v1/topology/{log_id}` – возвращает список узлов и групп топологии.<br>3. `GET /api/v1/node/{node_id}` – детали узла системы из лога.<br>4. `GET /api/v1/port/{node_id}` – получение портов узла системы из лога.<br>5. `GET /api/v1/log/{log_id}` – мета-информация о логе (status, количество узлов/портов, дата загрузки). |
| F-6  | Контейнеризация          | Docker Compose с сервисами `app` и `db`. Порт приложения – 8080 (может переопределяться переменной `PORT`).                                                                                                                                                                                                                                                               |

---

## 3. Нефункциональные требования

| Требование           | Описание                                                                                                                                      |
|----------------------|-----------------------------------------------------------------------------------------------------------------------------------------------|
| Код-стайл            | `go fmt`, lint-проверка.                                                                                                                      |
| Логирование          | Структурные логи в `stdout` (запрос, время парсинга, ошибки).                                                                                  |
| Настройки            | Через переменные окружения (`DATABASE_URL`, `PORT`, `LOG_LEVEL`).                                                                              |
| Документация         | `README.md` – как собрать, как запустить, примеры `curl`.                                                                                      |
| Отказоустойчивость   | В случае битых логов нужна обработка ошибок.                                                                                                   |

---

## 4. Как сдавать

1. Создайте репозиторий, сделайте его публичным.
2. В корне разместите `README.md` и `docker-compose.yml`.
3. Убедитесь, что `docker compose up -d` поднимает сервис без ошибок.
4. Будет плюсом создание `postman_collection` с тестами API.
5. Пришлите ссылку на репозиторий.
